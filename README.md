Агуулахын удирдлагын систем (Warehouse Management System)
Төслийн зорилго
Энэхүү системийн гол зорилго нь байгууллагын олон салбар агуулахын үйл ажиллагааг автоматжуулах, бараа материалын орлого, зарлагын хөдөлгөөнийг бүртгэх, үлдэгдлийг хянах болон тайлан мэдээ гаргах процессыг цэгцлэхэд оршино.

Үндсэн функцууд
Орлого ба Зарлагын бүртгэл: Бараа хүлээн авах болон гаргах бүрт үйлдэгдэх падааныг системээр үүсгэж, холбогдох мэдээллүүдийг (тоо ширхэг, огноо, хариуцагч этгээд) баримтжуулна.

Нөөцийн хяналт: Агуулах дахь бүх бараа эсвэл сонгосон тодорхой бараануудын одоогийн үлдэгдлийг цаг тухайд нь харах боломжтой.

Няравын тайлан: Тодорхой цаг хугацааны интервалд агуулахын эхний үлдэгдэл, нийт орлого, нийт зарлага болон эцсийн үлдэгдлийг нэгтгэн харуулна.

Тооллого ба Засвар: Барааны бодит үлдэгдлийг систем дэх тоотой тулгаж, илүүдэл эсвэл дутагдлыг бүртгэн, барааны үлдэгдлийг шинэчлэх боломжийг олгоно.

Классын зохиомж (Architecture)
Системийг дараах үндсэн классуудад хуваан зохион байгуулсан:

Warehouse (Агуулах): Системийн гол класс бөгөөд агуулахын нэр, байршил болон хариуцсан няравын мэдээллийг агуулна.

Product (Бараа): Барааны нэр, код, хэмжих нэгж зэрэг үндсэн атрибутуудыг тодорхойлно.

Accountant (Нярав): Агуулахын үйл ажиллагааг (орлого, зарлага, тооллого) удирдах функцуудыг гүйцэтгэнэ.

Transaction (Гүйлгээ - Падаан): InboundTransaction (Орлого) болон OutboundTransaction (Зарлага) гэсэн дэд хэлбэрүүдтэй байх бөгөөд гүйлгээний огноо, барааны жагсаалт, холбогдох хүмүүсийн мэдээллийг хадгална.

Inventory (Нөөц): Тухайн агуулах дахь бараа бүрийн одоогийн үлдэгдлийг хянах, тооллогын үр дүнгээр үлдэгдлийг шинэчлэх логикийг агуулна.

Report (Тайлан): Хугацаа болон барааны төрлөөр шүүлт хийж, нэгтгэл мэдээлэл боловсруулах үүрэгтэй.
